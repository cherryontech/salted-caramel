# Contributing
Before contributing to the repository, please discuss the proposed changes with the dev team before submitting them--whether through an issue, email, or another communication method.

We also have a **Code of Conduct** that all contributors are expected to follow in every interaction related to this project.

# Pull Request Process
1. Make sure to remove any installations or build dependencies before completing a build layer.
2. Update the `README.md` file to document any interface changes. This includes new environmental variables, exposed ports, file locations, and container parameters.
3. When opening a Pull Request, make sure you use the `pull_request_template.md` template to detail the changes and testing instructions.
4. A Pull Request can be merged by one of the developers once it has received approval from two or more reviewers. If you do not have merge permissions, you may request one of the reviewers to merge it on your behalf.

# Code of Conduct

## Our Commitment

We are dedicated to fostering an open, welcoming, and harassment-free environment for everyone regardless of age, body size, disability, ethnicity, gender identity or expression, experience level, nationality, appearance, race, religion, or sexual orientation.

## Our Standards

Positive contributions include:
- Using welcoming and inclusive language
- Respecting different viewpoints and experiences
- Accepting constructive feedback with grace
Prioritizing what benefits the community as a whole
- Showing empathy towards other members

Unacceptable behaviors include:
- Use of sexualized language, imagery, or unwelcome advances
- Trolling, personal attacks, or political insults
- Public or private harassment
- Sharing someone's private information (physical or electronic) without consent
- Any behavior that would be inappropriate in a professional environment

## Maintainer Responsibilities

Project maintainers (devs and PMs) are responsible for maintaining and upholding community standards. They may remove, edit, or reject comments, commits, code, and other contributions that violate this Code of Conduct. Maintainers also have the right to temporarily or permanently ban contributors who engage in inappropriate, threatening, or harmful behavior.

## Scope

This Code of Conduct applies within all project spaces and in public spaces when representing the project or community. Examples include posting about the project on an official social media account, or representing the project at an event.

## Enforcement

If you experience or witness abusive, harassing, or otherwise unacceptable behavior, please submit an issue. 

Project maintainers who fail to enforce this Code of Conduct in good faith may face temporary or permanent removal as determined by the project leadership.
